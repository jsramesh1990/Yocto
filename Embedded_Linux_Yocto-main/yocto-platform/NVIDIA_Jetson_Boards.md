# NVIDIA Jetson Boards + Yocto Workflow

NVIDIA Jetson platforms are widely used for:

* AI inference
* robotics
* autonomous systems
* video analytics
* edge AI
* CUDA workloads

Popular Jetson boards:

* Jetson Nano
* Jetson Xavier NX
* Jetson AGX Xavier
* Jetson Orin Nano
* Jetson AGX Orin

---

# 1. How NVIDIA Uses Yocto

NVIDIA officially provides:

* JetPack SDK
* L4T (Linux for Tegra)

Yocto support comes primarily through:

* `meta-tegra`

Official project:

* [meta-tegra GitHub](https://github.com/OE4T/meta-tegra?utm_source=chatgpt.com)

This layer integrates:

* CUDA
* TensorRT
* multimedia
* NVIDIA kernel
* boot firmware
* GPU drivers

# 12. Jetson Boot Architecture

Understanding boot is very important.

---

# Boot Flow

```text id="b9a7yb"
BootROM
   ↓
MB1
   ↓
MB2
   ↓
CBoot/UEFI
   ↓
Linux Kernel
   ↓
Device Tree
   ↓
RootFS
   ↓
systemd
```

Modern Jetson platforms increasingly use:

* UEFI
* secure boot
* A/B redundancy

# 13. Flashing Methods

Jetson flashing differs significantly from Raspberry Pi/i.MX.

Most flashing uses:

* USB recovery mode
* NVIDIA flashing tools

---

# Method 1 — NVIDIA SDK Manager

Official GUI flashing tool.

* [NVIDIA SDK Manager](https://developer.nvidia.com/nvidia-sdk-manager?utm_source=chatgpt.com)

Supports:

* JetPack install
* flashing
* CUDA setup

Useful for:

* development
* first-time setup

---

# Method 2 — `tegraflash.py` (Most Important)

Production flashing method.

Generated by Yocto/meta-tegra.

---

# Put Board Into Recovery Mode

Typical process:

1. hold RECOVERY button
2. press RESET
3. connect USB-C

Host detects:

```bash id="s1pbxq"
lsusb
```

Example:

```text id="u1x2e6"
NVIDIA Corp.
```

---

# Flash Board

```bash id="u5n3gf"
sudo ./doflash.sh
```

or:

```bash id="v4e0xs"
sudo ./flash.sh
```

Generated by:

* `meta-tegra`
* tegraflash package

---

# Method 3 — Initrd Flashing

Modern Orin platforms use:

* initrd flashing workflows

Advantages:

* faster
* flexible partitioning
* NVMe flashing

---

# Example NVMe Flash

```bash id="1ry3hm"
sudo ./initrd-flash.sh --external-device nvme0n1p1
```

Very common for:

* AI edge systems
* production devices

---

# Method 4 — SD Card Flashing

Mainly for:

* Jetson Nano
* Orin Nano devkits

Flash generated `.img` using:

```bash id="wnv3ow"
sudo dd if=image.img of=/dev/sdb bs=4M
```

or:

```bash id="vrv9z7"
sudo bmaptool copy image.img /dev/sdb
```

---

# Method 5 — Network Boot

Advanced enterprise setups use:

* NFS rootfs
* PXE-like workflows

Useful for:

* CI labs
* robotics testing

# 15. Verification Methods

This is where enterprise validation becomes important.

---

# Level 1 — UART Boot Verification

MOST IMPORTANT.

Use:

* USB-UART adapter

Monitor:

```bash id="0z6q8v"
picocom -b 115200 /dev/ttyUSB0
```

Verify:

* BootROM
* MB1/MB2
* UEFI
* kernel logs

---

# Level 2 — Linux Verification

Check:

```bash id="9v4k9d"
uname -a
```

```bash id="2b2r1g"
cat /etc/os-release
```

```bash id="l6s4yt"
dmesg
```

---

# Level 3 — GPU Verification

VERY IMPORTANT for Jetson.

Check GPU:

```bash id="w2z5jc"
nvidia-smi
```

(Available on some platforms)

Or:

```bash id="c9r0uv"
tegrastats
```

Monitor:

* GPU load
* thermal
* memory

---

# Level 4 — CUDA Verification

Test CUDA:

```bash id="ly3m7m"
deviceQuery
```

or:

```bash id="8u5e3h"
nvcc --version
```

---

# Level 5 — TensorRT Verification

Check AI acceleration:

```bash id="7n6j4q"
trtexec
```

Validate:

* inference performance
* GPU acceleration

---

# Level 6 — Multimedia Verification

Test:

* camera pipelines
* NVENC/NVDEC
* GStreamer

Example:

```bash id="3t7z0e"
gst-launch-1.0
```

---

# Level 7 — Camera Verification

Very important for robotics/AI.

Test:

* CSI cameras
* USB cameras

Examples:

```bash id="j7z5r2"
v4l2-ctl --list-devices
```

---

# Level 8 — AI Performance Validation

Benchmark:

* YOLO
* DeepStream
* TensorRT

Monitor:

* FPS
* latency
* thermal throttling

---

# Level 9 — Thermal + Stress Testing

Use:

```bash id="y4s2x8"

```

Monitor:

```bash id="q5d7r1"
tegrastats
```

Jetson systems often throttle under:

* poor cooling
* sustained AI workloads

# 20. Common Jetson Development Issues

| Problem                    | Cause              |
| -------------------------- | ------------------ |
| Recovery mode not detected | USB cable/port     |
| Flash failure              | wrong BSP version  |
| CUDA missing               | license/EULA       |
| Camera failure             | DTB mismatch       |
| Thermal throttling         | inadequate cooling |
| Boot loop                  | partition mismatch |
